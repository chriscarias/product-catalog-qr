# QR Code Setup Guide - IMPORTANT!

## ⚠️ Current Issue

The QR codes generated by the default implementation are **not scannable** because they use a simple pattern generator for demonstration purposes.

## ✅ Solution: Install a Real QR Code Library

You have **3 options** to generate real, scannable QR codes:

---

## Option 1: Using External API (Easiest - Requires Internet)

The updated `qr_generator.php` now uses the QR Server API by default, which generates real QR codes.

**What to do:**
1. Ensure your server has internet access
2. The QR codes will be automatically generated using the API
3. No additional installation needed!

**Limitations:**
- Requires internet connection
- Depends on external service availability

---

## Option 2: Install PHP QR Code Library (Recommended)

### Step 1: Download the Library

Download **phpqrcode** from: https://sourceforge.net/projects/phpqrcode/

OR use this direct link:
```
https://sourceforge.net/projects/phpqrcode/files/phpqrcode-2010100721_1.1.4.zip/download
```

### Step 2: Extract and Place Files

1. Extract the ZIP file
2. Copy the `phpqrcode.php` file to: `product-catalog/backend/lib/`
3. Create the lib directory if it doesn't exist:
   ```bash
   mkdir product-catalog/backend/lib
   ```

### Step 3: Update qr_generator.php

Replace the entire `product-catalog/backend/qr_generator.php` file with this code:

```php
<?php
require_once __DIR__ . '/lib/phpqrcode.php';

class QRCodeGenerator {
    
    public static function generate($data, $filename, $size = 300) {
        $qr_path = __DIR__ . '/../assets/qrcodes/' . $filename;
        
        // Generate real QR code using phpqrcode library
        QRcode::png($data, $qr_path, QR_ECLEVEL_L, 10, 2);
        
        return $filename;
    }
    
    public static function getQRUrl($data, $size = 300) {
        return "https://api.qrserver.com/v1/create-qr-code/?size={$size}x{$size}&data=" . urlencode($data);
    }
}
?>
```

### Step 4: Test

1. Go to Products in admin panel
2. Add a new product
3. View the product details
4. Scan the QR code with your phone - it should work!

## Option 3: Use Composer (For Advanced Users)

If you have Composer installed:

### Step 1: Navigate to backend folder
```bash
cd product-catalog/backend
```

### Step 2: Install library
```bash
composer require chillerlan/php-qrcode
```

### Step 3: Update qr_generator.php

Replace the entire file with:

```php
<?php
require_once __DIR__ . '/vendor/autoload.php';

use chillerlan\QRCode\QRCode;
use chillerlan\QRCode\QROptions;

class QRCodeGenerator {
    
    public static function generate($data, $filename, $size = 300) {
        $qr_path = __DIR__ . '/../assets/qrcodes/' . $filename;
        
        $options = new QROptions([
            'version'    => 5,
            'outputType' => QRCode::OUTPUT_IMAGE_PNG,
            'eccLevel'   => QRCode::ECC_L,
            'scale'      => 10,
            'imageBase64' => false,
        ]);
        
        $qrcode = new QRCode($options);
        $qrcode->render($data, $qr_path);
        
        return $filename;
    }
    
    public static function getQRUrl($data, $size = 300) {
        return "https://api.qrserver.com/v1/create-qr-code/?size={$size}x{$size}&data=" . urlencode($data);
    }
}
?>
```

## Quick Fix for Existing Products

If you already created products with non-scannable QR codes:

### Option A: Regenerate All QR Codes

Run this script once (create as `backend/regenerate_qr.php`):

```php
<?php
require_once '../config/database.php';
require_once 'qr_generator.php';

$conn = getConnection();
$result = $conn->query("SELECT id FROM products WHERE qr_code IS NOT NULL");

while ($product = $result->fetch_assoc()) {
    $product_id = $product['id'];
    $product_url = FRONTEND_URL . '/product.php?id=' . $product_id;
    $qr_filename = 'product_' . $product_id . '.png';
    
    // Delete old QR
    $old_path = __DIR__ . '/../assets/qrcodes/' . $qr_filename;
    if (file_exists($old_path)) {
        unlink($old_path);
    }
    
    // Generate new QR
    QRCodeGenerator::generate($product_url, $qr_filename);
    
    echo "Regenerated QR for product ID: $product_id<br>";
}

$conn->close();
echo "Done!";
?>
```

Then visit: `http://localhost/product-catalog/backend/regenerate_qr.php`

### Option B: Delete and Re-add Products

1. Delete existing products
2. Re-add them (QR codes will be generated with new library)

## Troubleshooting

### "Class 'QRcode' not found"
- You didn't install the phpqrcode library correctly
- Make sure `phpqrcode.php` is in the `backend/lib/` folder
- Check the require_once path in qr_generator.php

### "Cannot write to file"
- The `assets/qrcodes/` directory is not writable
- Fix permissions:
  ```bash
  chmod 755 assets/qrcodes/
  ```

### QR codes still not scanning
- Make sure you regenerated the QR codes after installing the library
- The old QR codes are still the demo ones
- Run the regenerate_qr.php script above
